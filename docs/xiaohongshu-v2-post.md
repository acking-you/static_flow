# 小红书推广文案 v2

朋友说想听一首歌，2 分钟后就能在线听了

---

朋友在我博客评论区写了句「想听周杰伦的晴天」。

2 分钟后，歌已经在我的音乐库里了，在线就能播，还带歌词同步。

他问我：你手速这么快？

不是我快，是 AI 帮我干的。

我用 Rust 全栈造了个系统叫 StaticFlow，最开始是个 AI 知识博客，最近加了音乐模块，变成了一个本地优先的个人内容平台。

最好玩的功能是「许愿点歌」。

朋友说想听一首歌，我说：去评论区写个许愿。他写了「想听周杰伦的晴天」，然后：

1. 我在管理面板点一下审核通过
2. Codex 自动调用 music-ingestion Skill
3. 自动搜索 B站 + 网易云，找到最佳音源
4. 下载、生成元数据、向量嵌入，写入 LanceDB
5. 评论自动回复「已入库，去听吧」

全程不到 2 分钟，除了审核那一下，零人工干预。

朋友：？？？这也行？

技术上怎么做到的：
- Codex + Skill 文件：AI 加载一个 Markdown 就知道该怎么搜歌、下载、入库，不需要写框架代码
- 双源搜索：B站音频提取 + 网易云 API，覆盖面广
- LanceDB 向量搜索：BGE-Small 双语模型生成嵌入，FTS + 语义混合检索，RRF 融合排序
- 在线播放：Yew/WASM 前端，全局播放器 + 歌词同步滚动
- 本地存储：所有音乐文件和数据都在本地，不依赖任何云服务

2.0 新增的不只是音乐：
- 音乐库：搜索、播放、歌词、统计
- 许愿点歌：评论驱动的 AI 自动入库
- 音乐评论：每首歌都能讨论
- 播放统计：热门排行、播放追踪

1.0 的能力全部保留：文章发布、AI 评论、混合搜索、双语翻译、Skill 工作流。

说实话，做这个项目最大的感受是：Skill 这个模式太对了。Anthropic 说的 "Don't build agents, build skills instead" 不是空话——每个 Skill 就是一个 Markdown 文件，AI 读完就能干活，不需要写一行胶水代码。现在已经有 10+ 个 Skill 覆盖从写文章到搜歌的全链路。
